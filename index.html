「Webコンテンツ作成」の成果

本テーマの学習目的

本テーマの学習目標は、全6回授業を通してHTML、CSS,JavaScript,DOM,イベント処理などのWebページを作る文書構造を学ぶことである。 またそれらの知識を利用して、簡易的なWebサイトを作成することが本テーマの最終目標である。

プロジェクト課題について

プロジェクト課題ではグルメ情報を選択した。利用者がプルダウンメニューの17種類のジャンルから一つ選び、その選んだメニューを検索、表示する。 検索ボタンを押すことで、ジャンルごとに近くのお店の情報をを通信して入手する。 表示する内容は、店舗名、キャッチコピー、アクセス、最寄駅、住所、予算、営業日時、特徴の8つである。 ページの検索は何度でも可能で、ジャンルを変更し、検索ボタンを押すことでページは更新される。その際、前のジャンルの情報の表示が残ることはありません。

  • グルメ情報

数当てゲーム

この課題は、まず答えとして1から10の数を生成する。つぎに回答者はその答えを予想し、それをテキストボックスに入力して、ページの解答ボタンをクリックして判定を開始する。 判定結果が正解か不正解かを判断し、それを表示する。 もし不正解なら、入力された数より生成された答えが大きいか小さいかを判断し、ユーザにヒントを与える。 解答権は3回まで。 またDOMを利用して、結果をコンソールではなく、ページに表示するよう変更。 テキストへの書き込みは上書きされるので、メソッドの1〜3回目の呼び出し結果は確認ない。4回目の結果のみ表示される。 最後の課題では解答の入力欄、またその横に「回答」と書かれたボタンを設置した。加えて、回答ボタンを押すたびにhantei()メソッドを1回ごとに実行できるように変更した。 またコンソールではデッバク用の答えが常に確認できるようになっている。 そしてこの一連の動きが、ページに反映できるように変更した。

第1回 WWW と HTML

今回はHTMLの使い方について学んだ。 HTMLはWebページの構造を記述するための書式であり。HTML文書を作ることによって、Webページに表示するコンテンツと、ページのレイアウト作ることができるようになる。 要素にはブロック要素とインライン要素があり、インライン要素の中にブロック要素は配置できない。また要素には属性という付加情報をつけることができる。 属性にはclass,id,imgがあり、これはCSSなどでページを装飾するときに、見た目を柔軟に変更するために使用したりすることができる。

HTMLを使うためのお約束や要素を学んだ。HTMLの使い方を調べることが可能になった。そしてデバックが簡単にできる方法を学んだ。 また「www」のことや、URLのちょっとした仕組みについても学んだ。加えて、Githubの使い方について復習した。また次の課題に取り組んだ。

第3回 JavaScript の基本

JavaScriptはWebページ上で複雑な機能を作るプログラミング言語である。ページを開いた直後や、ボタンをクリックした後、プログラムが実行される。 ページの自動更新やアニメーションなど動きのあるページを実現するためにJavaScriptは利用できる。 またJavaScriptはインタプリタ言語である。プログラムは1行ずつ読み取りながら実行される。なのでJavaにおけるコンパイルという手間がない。 しかし欠点もある。それは、エラーが起こっていても気づかないことがあったり，エラーの原因を見つけにくいという短所である

JavaScriptプログラムを書く方法は複数ある。 HTMLではscript要素の中にJavaScriptプログラムを作成する方法。だがこれは推奨されない。 HTMLファイルとは別にJavaScriptファイルを作成する方法。これは推奨される。

今回から、Firefoxのウェブ開発ツールを使用し、新しく2つの方法でデバッグできるようになった。 エラー発生時にプログラムの実行を一時停止する、またはブレークポイントを設定してプログラムの実行を一時停止するである。 Javaではエラーになるのに、JavaScriptではエラーにならないプログラムはかなりある。例えば、引数が足りない関数の呼び出しなどである。 それらを見つけるのに、この2つの方法は大いに役に立つだろう。

第4回 DOM と JavaScript

DOMはWebコンテンツを表現するデータ構造である. DOMを使用することで、JavaScriptなどによって、Webコンテンツの内容を読み取ったり、コンテンツに変更を加えたりする処理が書けるようになる。 DOMとJavaScriptは組み合わせることができ、それにより動きのあるWebページを作成できるようになる。

DOMの各ノードはJavaScriptにおいて、オブジェクトとして利用可能である。 また、取得したオブジェクトに変更を加えたり、新しいオブジェクトを追加することでDOMの構成に変更を加えることができる、この変更によってページのコンテンツや見た目が変化する。 DOMは単体で使用することは少ない。イベント処理と組み合わせて利用すると実用性が高い。

DOMプログラムは常に、1.要素の検索 2.検索した要素に対して何かすることを記述する。 またそれもいくつかパターンがある。1の要素の検索においては、まず要素の検索は1つずつ行うのか、複数まとめて行うのか。 2の何かする場合においては、要素のテキストやCSSを設定したり、新し要素を設定したり、また新しい要素を内容とする別の新しい要素を作成したりなどである。 DOMを使ったプログラムを作るのは難しいので、コンソールを使ってプログラムを作るのが良い。1行ずつ確認ができるため、ミスに気づきやすい。

第5回 イベント処理

イベントはWebページ上で起こった動作や出来事のことである。イベント発生時にJavaScriptプログラムが動作するような処理をイベント処理という。 それにより、数当てゲームにはテキストボックスを設置し、ゲームの結果も1回ごとに表示可能になった。 またボタンを押すことで背景の色を変えたりする、クリックするとボックスの色を変えたりすることが可能になった。

イベント処理は　1.イベントハンドラの登録　2.イベントハンドラの定義の順で行われる。 イベント処理の動きを要約すると「要素 〇〇 を △△ したら，□□ を実行する」である。 〇〇には検索した要素が入り、△△ にはイベント名などが入り、□□には関数などを定義する。

今回から、input要素を利用することで、Webサイトでよく見るユーザインタフェースであるラジオボタンとチェックボックス作れるようになった。 これらによってより動きのあるwebページが作れるようになった。

工夫点，苦心点，強調点，感想

作成したレポートページで工夫したことは配色である。一般的に赤やオレンジ、黄色などの暖色系などは誘目性が高く、逆に青や緑、紫などの寒色系は誘目性が低いとされている。 そこに着目し、誘目性の低い色をWebページに採用し、ページを注目ポイントを文字の大きさを変更することだけに絞った。 なぜなら個人的に誘目性の強い色が多く利用されているサイトは、目が痛くなるので見ていたいと思えないと感じていた。逆にタイトルでしか誘目性の色を利用しないような、利用回数の少ないサイトであっても、 本文を読んでいる時にチラチラと目に入ってきて不快感を感じていたからだ。 今回の配色によって、利用者の目の負担を減らし、強く注意を引くようなものがないので、ページ全体になにがあるのかざっくりとわかるようなレポートページに仕上がった。

プロジェクト課題では検索のしやすさを重視した。その上で特にジャンルを選択するGUIは悩んだ。 それぞれのメリットをもう一度よく考えた上で、プルダウンメニューの操作する時にだけ表示され、画面上の領域を占領しないメリットが、ジャンルの数が17種と多いグルメ情報においてとても大きいと感じたので採用した。 実際、検索時にジャンルを選択ミスすることがないので、使いやすくなったといえる。またジャンル指定時に入力をしないので、バグを減らすこともできただろう。 また店名表示をh3の表示で作成することで、どこの情報を見ているかを分かりやすくした。

今回、Webページ作成をしてみての感想は、Webページを作ることは時間がかかるし難しいである。 いままでのプログラミング系の課題は、JavaならJavaでできるものばかりであったため、HTML,JavaScript、CSSなど多くの言語を組み合わせて利用することに驚いた。 複数の言語を組み合わせて利用する感覚がいままでの感覚と大きく異なっていたので、理解するのに時間がかかった。 しかしながら、授業のたびにボリュームアップしていく自分のページを見ることでやりがいを感じることができた。 また、私たちが何気なく利用しているサイトは多くの工夫が詰め込まれていると改めて感じた。特に、ゲームの公式サイトはとてもスタイリッシュな動きを持っているので相当お金がかかっているんだろうと感じた。 今までの授業を通して、Webページを作成するものとしての視点を入れることができた。 その視点を利用して、今後はただの閲覧者に収まらないようなWebページ利用者になっていきたい。そのためには、手にいれた視点を発展させるために、このテーマが終わった後も新たな知識の習得をしていきたい。
